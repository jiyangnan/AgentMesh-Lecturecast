COMMUNITY LOCAL · DIRECTOR OPTIONAL

하나의 아이디어를, 강의 영상 한 편으로.

Lecturecast Community 는 완전 로컬·오픈소스 영상 제작 워크플로입니다. 주제를 AI 에이전트에게 넘기면 당신의 머신에서 대본, 더빙, 렌더링을 처리하고 Bilibili 가로판과 Xiaohongshu 세로판을 동시에 출력합니다. 더 명확한 창작 방향이 필요하면 선택형 Director 가 구조화된 선택을 서명된 제작 계획으로 바꾸며, 원본 미디어와 전체 미디어 파이프라인은 계속 로컬에 남습니다. (Bilibili 와 Xiaohongshu 는 중국의 주요 영상 플랫폼—각각 가로판/세로판. 사용하지 않더라도 두 포맷을 함께 출력합니다.)

Community지금 사용 가능

완전 로컬. 영구 무료.

계정이나 LectureCast API 키가 필요 없습니다. 에이전트가 대본, 더빙, 자막, 모션, 이중 포맷 렌더링, 커버를 당신의 머신에서 처리하며 원본 콘텐츠를 LectureCast 로 보내지 않습니다.

Director단계적 액세스

클라우드 디렉팅, 로컬 제작.

에이전트 네이티브 선택 UI 로 대상, 서사, 비주얼, 전달 방식을 결정합니다. 클라우드는 서명된 ProductionManifest 를 반환하고, credit 은 생성 확인 후에만 사용됩니다. 미디어, 더빙, 편집, 렌더링은 로컬입니다.

~ run-this-locally 
$ curl -fsSL https://raw.githubusercontent.com/jiyangnan/AgentMesh-Lecturecast/main/scripts/install.sh | bash
$ lecturecast workflow      # 로컬 워크플로 위치를 출력해 AI 에이전트에게 전달

# 에이전트는 AGENTS.md / docs/LOCAL-WORKFLOW.md 에 따라 로컬에서 엔드 투 엔드 실행:
# 대본 → 더빙(edge-tts / MiniMax) → Remotion 렌더링 → ffmpeg 자막 굽기 → 이중 플랫폼 영상
왜 맡겨야 하는가

대본은 AI 에게 맡길 수 있어도, 제작이 진짜 고통입니다.

AI 에게 대본을 쓰게 하는 건 어렵지 않습니다. 어려운 건 그 뒤—더빙, 타임라인 정렬, 렌더링, 자막 굽기, 커버 작업, 그리고 가로판과 세로판을 매번 따로 맞추는 일입니다.

Lecturecast 는 이 제작 파이프라인 전체를 당신의 AI 에이전트의 손에 통째로 넘깁니다. 전부 당신의 머신에서 실행되며, 에이전트가 텍스트가 아니라 "완성된 영상"을 실제로 제공합니다.

기능 · 모두 로컬 동작

하나의 주제에서 두 편의 영상으로.

i.

AI 가 쓰는 강의 대본

주제를 에이전트에게 넘기면 구조화된 강의 대본—섹션 구분, 논리적 흐름, 구어체—을 생성합니다. 대본은 승인 게이트: 당신이 확인하고 수정한 뒤에야 제작에 들어갑니다.

ii.

로컬 더빙과 정렬

대본은 로컬에서 자연스러운 음성으로 변환되고, 문장 단위로 자동 분할되어 타임라인에 정렬됩니다. 긴 문단은 길이 검증과 자동 재시도로 음성과 화면이 어긋나지 않습니다. 기본은 무료 Edge TTS, 본인의 MiniMax key 로 업그레이드 가능합니다.

iii.

이중 플랫폼 렌더링

같은 대본을 로컬 Remotion 으로 Bilibili 가로판(16:9)과 Xiaohongshu 세로판(9:16)으로 한 번에 출력합니다. 각각의 레이아웃에 최적화되어, 다시 편집할 필요가 없습니다.

iv.

자막 + 커버

로컬 ffmpeg 가 중국어 자막을 굽고, 양 플랫폼용 커버를 생성합니다. Xiaohongshu 의 컴플라이언스 체크도 자동 실행—받자마자 게시 가능, 편집 도구를 열 필요가 없습니다.

로컬 3 가지 세트

모두 당신의 머신에서.

클라우드 미디어 워커도, 원격 렌더러도 없습니다. Director 를 선택해도 클라우드는 창작 결정과 서명된 Manifest 만 담당합니다. 아래 세 가지를 설치하면 에이전트가 미디어 제작을 로컬에서 엔드 투 엔드 실행할 수 있습니다. install.sh 가 누락된 것을 알려 줍니다.

RENDER

Node + Remotion

코드 기반 애니메이션으로 두 비율을 렌더링. brew install node 후 npm install.

VOICE

Python + edge-tts

로컬 더빙, 문장 분할, 타임라인 정렬. 기본은 무료 Edge TTS, MiniMax key 로 업그레이드 가능.

MUX

ffmpeg(libass 포함)

중국어 자막을 굽고, 최종 영상을 합성. brew install ffmpeg 로 libass 포함 빌드 설치.

자주 묻는 질문

FAQ.

유료인가요? 계정 가입이 필요한가요?
Community 는 영구 무료이며 계정이나 LectureCast API 키가 필요 없습니다. 선택형 Director 는 단계적 액세스가 시작된 뒤 AgentMesh360 계정과 API 키가 필요합니다. 선택과 Brief 작성은 무료이고, ProductionManifest 생성을 확인한 경우에만 고정 credit 이 차감됩니다.
영상이 클라우드로 업로드되나요?
아니요. Community 는 LectureCast 서비스에 아무것도 보내지 않습니다. Director 를 선택하면 제한된 소스 요약, 안정적인 선택 ID, Brief, 클라이언트 기능만 클라우드로 전송됩니다. 원본 미디어, 더빙, 자막, 편집, 렌더링, 완성본은 항상 로컬에 남습니다.
더빙은 무엇을 쓰나요? 더 좋은 음색으로 바꿀 수 있나요?
기본은 무료 Edge TTS—바로 사용할 수 있고 출시에는 충분합니다. 더 따뜻한 음색이 필요하다면, 본인의 MiniMax key 를 가져올 수 있습니다—이는 Lecturecast 제공이 아니라 당신이 직접 minimaxi.com 에 등록하는 서드파티 서비스입니다. key 는 환경 변수 MINIMAX_API_KEY 로만 전달되며, 디스크에 기록되지 않습니다.
영상은 완전 자동 생성인가요? 수정할 수 있나요?
대본은 반드시 먼저 멈춰서 당신의 확인과 수정을 받은 후에야 제작에 들어갑니다—내용은 당신이 통제하고, AI 가 제작을 담당합니다.
무엇을 설치해야 하나요?
로컬 3 가지 세트: Node + Remotion(렌더링), Python + edge-tts(더빙), libass 포함 ffmpeg(자막 굽기). install.sh 가 누락된 것을 알려 줍니다. macOS 라면 brew install node ffmpeg 로 충분합니다.
어떤 플랫폼용 영상에 대응하나요?
Bilibili 가로판(1920×1080)과 Xiaohongshu 세로판(1080×1920)을 동시에 출력합니다. 중국어 자막 굽기 완료, 커버 생성 완료입니다.
정말 오픈소스인가요?
예. Apache 2.0. 코드는 모두 GitHub 에 있습니다. 자유롭게 포크, 개조, issue 등록을 환영합니다.
