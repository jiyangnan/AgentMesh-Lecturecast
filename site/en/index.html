Turn one idea into a course video.

Lecturecast Community is a fully local, open-source video-production workflow: hand a topic to your AI agent and it scripts, voices, and renders both Bilibili landscape and Xiaohongshu portrait videos on your machine. Why hand it off

AI can write a script. Production is the actual pain.

Getting an AI to write a script is easy. What's hard is everything after — voiceover, timeline alignment, rendering, subtitle burn-in, cover design, and producing landscape + portrait versions each time.

Lecturecast hands the whole production pipeline to your AI agent. It runs on your own machine end-to-end, so the agent actually delivers finished videos — not just text.

Capabilities · all local

From one topic to two finished videos.

i.

AI-written script

Hand the topic to your agent and it produces a structured course script — sectioned, logical, in spoken language. The script is the approval gate: you confirm or edit before production starts.

ii.

Local voiceover and alignment

The script becomes natural speech on your machine, auto-segmented and time-aligned. Long segments are length-checked and retried automatically so audio and video stay in sync. Edge TTS is free by default; MiniMax is optional via your own key.

iii.

Dual-platform rendering

One script, rendered locally with Remotion into both Bilibili landscape (16:9) and Xiaohongshu portrait (9:16), each with its own layout. No second pass of editing required.

iv.

Subtitles + covers

Local ffmpeg burns in Chinese subtitles and generates dual covers. A Xiaohongshu compliance check runs automatically so you can post immediately, no editing software needed.

Local three-piece stack

All on your own machine.

No cloud media workers and no remote renderer. Even with Director, the cloud handles only creative decisions and a signed Manifest. Install these three, and your agent can run media production end-to-end on this machine. install.sh checks what's missing.

RENDER

Node + Remotion

Code-driven animated scenes that render both aspect ratios. brew install node then npm install.

VOICE

Python + edge-tts

Local voiceover, sentence segmentation, and timing alignment. Free Edge TTS by default; upgrade with your own MiniMax key.

MUX

ffmpeg (with libass)

Burns Chinese subtitles and stitches the final video. brew install ffmpeg gives you a libass-enabled build.

FAQ

FAQ.

Does it cost anything? Do I need an account?
Community is free forever and needs no account or LectureCast API key. Optional Director will require an AgentMesh360 account and API key when staged access opens. Choices and Brief creation are free; only a confirmed ProductionManifest generation deducts fixed credits.
Does the video get uploaded to the cloud?
No. Community sends nothing to a LectureCast service. If you choose Director, only a bounded source summary, stable option IDs, the Brief, and client capabilities go to the cloud. Original media, voice, subtitles, editing, rendering, and finished outputs always stay local.
What does the voiceover use? Can I get a richer voice?
By default it uses the free Edge TTS — works out of the box and is good enough to ship. For a warmer voice you can bring your own MiniMax key — that's a third-party service you register for yourself, not provided by Lecturecast. The key is passed via the MINIMAX_API_KEY environment variable and is never written to disk.
Are the videos generated fully automatically? Can I edit?
The script always pauses for your confirmation and edits before production starts. You own the content; the AI handles the production.
What do I need to install?
Three local pieces: Node + Remotion (render), Python + edge-tts (voiceover), and ffmpeg with libass (subtitle burn-in). install.sh tells you what's missing; on macOS, brew install node ffmpeg covers it.
Which platforms are the outputs for?
Both Bilibili landscape (1920×1080) and Xiaohongshu portrait (1080×1920) at once, both with burned-in subtitles and a generated cover.
Is it actually open source?
Yes. Apache 2.0. All the code is on GitHub — fork, modify, file issues.
