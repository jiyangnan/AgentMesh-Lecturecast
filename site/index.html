OPEN SOURCE · 完全本地运行

把一个想法，变成一条课程视频。

Lecturecast 是一套完全本地、开源的出片工作流：把一个主题交给你的 AI agent，它在你自己的机器上跑完整条流水线——写讲稿、配音、渲染，同时产出 B 站横版和小红书竖版，烧好中文字幕、配好封面。不需账户、不上云、免费、Apache 2.0 开源。

为什么不自己剪

写稿能交给 AI，出片才是真麻烦。

让 AI 写个脚本不难，难的是后面——配音、对轴、渲染、烧字幕、做封面、还要横竖两版各适配一次。

Lecturecast 把这条出片流水线整个交到你的 AI agent 手里，全程在你自己的机器上跑，让它真的能交付成片，而不是只给你一段文字。

产品能力 · 全在本地跑

从一句主题，到两条成片。

i.

AI 写讲稿

把主题交给你的 agent，它生成结构化课程讲稿——分节、有逻辑、口语化。脚本是审批关卡，先给你确认和修改，再进入制作。

ii.

本地配音对轴

讲稿在本机转成自然语音，自动切句、对齐时间轴，长段落自动校验时长重试，不会出现声画错位。默认 Edge TTS 免费，可选自带 MiniMax。

iii.

双平台渲染

同一份内容，用本机的 Remotion 一次产出 B 站横版（16:9）和小红书竖版（9:16），各自适配排版，不用你二次剪辑。

iv.

字幕 + 封面

本机 ffmpeg 自动烧录中文字幕、生成双平台封面，并跑一遍小红书合规检查。拿到就能直接发，不用再开剪辑软件。

一次生成 · 双版本

B 站和小红书，各发各的。

同一个主题、同一份讲稿，在本地一次跑完同时产出两个比例的成片，分别适配横屏与竖屏的排版和字幕，直接发布即可。

1920 × 1080

B 站横版

16 : 9

适合知识讲解、长视频；Remotion 渲染的清晰排版 + 烧录字幕。

1080 × 1920

小红书竖版

9 : 16

适合手机竖屏、信息流；Remotion 渲染的竖版排版 + 封面。

本地三件套依赖

全部跑在你自己的机器上。

没有云端 worker，没有远程渲染。装好这三样，你的 agent 就能在本机端到端出片。install.sh 会帮你检查缺哪样。

RENDER

Node + Remotion

用代码驱动的动画场景渲染双平台画面。brew install node 后 npm install 即可。

VOICE

Python + edge-tts

本地配音、切句、对轴。默认免费 Edge TTS；想升级音色就自带 MiniMax key。

MUX

ffmpeg（带 libass）

烧录中文字幕、合成成片。brew install ffmpeg 一行装好带 libass 的版本。

常见问题

FAQ.

要钱吗？要注册账户吗？
都不要。Lecturecast 是完全开源（Apache 2.0）的本地工具，没有账户、没有云服务、没有 key，免费使用。所有处理都在你自己的机器上完成。
视频会上传到云端吗？
不会。写稿、配音、渲染、烧字幕全程在本机跑，不需要联网渲染，你的内容不会离开你的电脑。
配音用什么？能换更好的音色吗？
默认用免费的 Edge TTS，开箱即用、足够出片。如果想要更暖的音色，可以自带（BYOK）一个 MiniMax key —— 那是你自己在 minimaxi.com 注册的第三方服务，不是 Lecturecast 提供的 key，只通过环境变量 MINIMAX_API_KEY 传入、绝不写入磁盘。
视频是全自动生成的吗？我能改吗？
脚本会先给你确认和修改（审批关卡），满意后才进入配音和渲染——你掌控内容，AI 负责出片。
需要装什么？
本地三件套：Node + Remotion（渲染）、Python + edge-tts（配音）、ffmpeg 带 libass（烧字幕）。install.sh 会帮你检查缺哪样，macOS 用 brew install node ffmpeg 即可。
支持哪些平台的成片？
一次产出 B 站横版（1920×1080）和小红书竖版（1080×1920）两个版本，已烧录中文字幕并生成封面。
真的开源吗？
是。整套工具 Apache 2.0 开源，代码全在 GitHub，欢迎自取、改造、提 issue。
