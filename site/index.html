为什么不自己剪

写稿能交给 AI，出片才是真麻烦。

让 AI 写个脚本不难，难的是后面——配音、对轴、渲染、烧字幕、做封面、还要横竖两版各适配一次。

Lecturecast 把这条出片流水线整个交到你的 AI agent 手里，全程在你自己的机器上跑，让它真的能交付成片，而不是只给你一段文字。

产品能力 · 全在本地跑

从一句主题，到两条成片。

i.

AI 写讲稿

把主题交给你的 agent，它生成结构化课程讲稿——分节、有逻辑、口语化。脚本是审批关卡，先给你确认和修改，再进入制作。

ii.

本地配音对轴

讲稿在本机转成自然语音，自动切句、对齐时间轴，长段落自动校验时长重试，不会出现声画错位。默认 Edge TTS 免费，可选自带 MiniMax。

iii.

双平台渲染

同一份内容，用本机的 Remotion 一次产出 B 站横版（16:9）和小红书竖版（9:16），各自适配排版，不用你二次剪辑。

iv.

字幕 + 封面

本机 ffmpeg 自动烧录中文字幕、生成双平台封面，并跑一遍小红书合规检查。拿到就能直接发，不用再开剪辑软件。

一次生成 · 双版本

B 站和小红书，各发各的。

同一个主题、同一份讲稿，在本地一次跑完同时产出两个比例的成片，分别适配横屏与竖屏的排版和字幕，直接发布即可。

本地三件套依赖

全部跑在你自己的机器上。

正式支持 macOS 与 Windows，不支持 Linux/WSL。没有云端媒体 worker，也没有远程渲染。云端只负责商业权限、创作决策与签名 Manifest；install.sh / install.ps1 会检查本机缺少什么。

RENDER

Node + Remotion

用代码驱动的动画场景渲染双平台画面。安装 Node 20+ 后由 npm install 准备项目依赖。

VOICE

Python + edge-tts

本地配音、切句、对轴。默认免费 Edge TTS；想升级音色就自带 MiniMax key。

MUX

ffmpeg（带 libass）

烧录中文字幕、合成成片。macOS 使用 ffmpeg-full；Windows 使用带 libass 的原生 ffmpeg build。最终都由 doctor 按能力验证。

怎么用

设计为 agent-friendly。

核心是一套本地工作流，把它交给你的 AI 助理。脚本生成后会停下来等你确认，确认满意才进入配音和渲染。

01

装上 CLI

macOS 使用 install.sh，Windows PowerShell 使用 install.ps1；两者都会安装商业 CLI、签名验证和 Agent Skill。接着运行 lecturecast auth login 与 lecturecast onboard --json。

常见问题

FAQ.

要钱吗？要注册账户吗？
需要有效的 AgentMesh360 付费账户和通用 API Key。Lecturecast 使用现有共享 credits，无需另购独立订阅；结构化选择和 Brief 不扣费，只有你确认生成 ProductionManifest 时才固定扣除 10 credits。
视频会上传到云端吗？
原始媒体不会上传。只有受限素材摘要、稳定选项 ID、Brief 和客户端能力进入云端 Director；原始媒体、配音、字幕、编辑、渲染与成片始终留在本机。
配音用什么？能换更好的音色吗？
默认用免费的 Edge TTS，开箱即用、足够出片。如果想要更暖的音色，可以自带（BYOK）一个 MiniMax key —— 那是你自己在 minimaxi.com 注册的第三方服务，不是 Lecturecast 提供的 key，只通过环境变量 MINIMAX_API_KEY 传入、绝不写入磁盘。
视频是全自动生成的吗？我能改吗？
脚本会先给你确认和修改（审批关卡），满意后才进入配音和渲染——你掌控内容，AI 负责出片。
需要装什么？
本地三件套：Node + Remotion（渲染）、Python + edge-tts（配音）、ffmpeg 带 libass（烧字幕）。macOS 的 install.sh 与 Windows 的 install.ps1 都会区分 CLI 安装成功和渲染器就绪；以 doctor 的能力检查为准。
支持哪些平台的成片？
一次产出 B 站横版（1920×1080）和小红书竖版（1080×1920）两个版本，已烧录中文字幕并生成封面。
真的开源吗？
是。整套工具 Apache 2.0 开源，代码全在 GitHub，欢迎自取、改造、提 issue。
